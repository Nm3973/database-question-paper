Triggers

Introduction

Triggers are special types of stored procedures that are automatically executed or fired when certain events occur in a database. They are used to enforce business rules, maintain data integrity, and provide audit trails.

Types of Triggers

  • Before Triggers: Execute before an insert, update, or delete operation.
  • After Triggers: Execute after an insert, update, or delete operation.
  • Instead of Triggers: Used on views to perform an insert, update, or delete operation instead of the standard action.

Trigger Syntax

CREATE TRIGGER trigger_name 
{BEFORE | AFTER} {INSERT | UPDATE | DELETE} 
ON table_name 
FOR EACH ROW 
BEGIN 
    ... 
END;

Example Trigger

CREATE TRIGGER update_student_grade
AFTER UPDATE ON students
FOR EACH ROW
BEGIN
    IF NEW.grade <> OLD.grade THEN
        INSERT INTO grade_changes(student_id, old_grade, new_grade, change_date)
        VALUES (OLD.id, OLD.grade, NEW.grade, NOW());
    END IF;
END;

Use Cases for Triggers

Triggers can be used for various purposes, including:

  • Enforcing business rules and constraints.
  • Automating system tasks, such as updating related tables.
  • Maintaining audit trails by logging changes to sensitive data.
  • Synchronizing data across multiple tables.
