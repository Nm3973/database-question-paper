Fragmentation and Lossless-Join

Introduction

Fragmentation in a database refers to dividing a database into smaller pieces called fragments, which can be managed and accessed separately. A lossless-join ensures that these fragments can be recombined to reconstruct the original database without losing any data.

Types of Fragmentation

Horizontal Fragmentation

Horizontal fragmentation divides a table into subsets of rows, each subset being a fragment. This is typically based on a condition applied to the rows.
