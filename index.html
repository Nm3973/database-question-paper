  17. Answer: A trigger is a procedural code that is automatically executed in response to certain events on a particular table or view. Triggers can enforce business rules, ensure data integrity, and provide audit trails.
    Syntax: 
    CREATE TRIGGER trigger_name 
{BEFORE | AFTER} {INSERT | UPDATE | DELETE} 
ON table_name 
FOR EACH ROW 
BEGIN 
    ... 
END;

    Example: 
    CREATE TRIGGER update_student_grade
AFTER UPDATE ON students
FOR EACH ROW
BEGIN
    IF NEW.grade <> OLD.grade THEN
        INSERT INTO grade_changes(student_id, old_grade, new_grade, change_date)
        VALUES (OLD.id, OLD.grade, NEW.grade, NOW());
    END IF;
END;

    A table is in 3NF if it is in 2NF and all the attributes are functionally dependent only on the primary key. Steps:
    1. Identify the candidate keys of the table.
    2. Ensure that all non-key attributes are fully functionally dependent on the primary key.

  19. Answer:
    • Impedance Mismatch: The difference in data models between SQL (relational) and programming languages (object-oriented) creates complexity in data translation.
    • Error Handling: Properly catching and managing SQL errors within application code.
    • Query Optimization: Writing efficient queries to avoid performance bottlenecks.
    • Security Concerns: Protecting against SQL injection attacks.
    • Connection Management: Efficiently managing database connections to ensure application performance.
    • Concurrency Control: Handling concurrent data access to avoid deadlocks and ensure data consistency.
    • Transaction Management: Ensuring ACID properties (Atomicity, Consistency, Isolation, Durability).
    • Schema Evolution: Adapting application code to changes in the database schema.
  20. Answer:
    • Super Key: A set of one or more columns that uniquely identify a record in a table.
    • Candidate Key: A minimal super key, with no unnecessary columns.
    • Primary Key: A candidate key chosen to uniquely identify records in a table.
    • Alternative Key: A candidate key not chosen as the primary key.
    • Foreign Key: A column or set of columns that establish a relationship between two tables, referencing the primary key in another table.

    Example:
    Student ID (PK) Name Course ID (FK)
    1 John 101
    2 Jane 102
    3 Doe 103
  21. Answer: Conflict Equivalent Transactions: Two schedules are conflict equivalent if they involve the same operations and the order of conflicting operations is the same.
    Conflict Serializability: A schedule is conflict serializable if it can be transformed into a serial schedule by swapping non-conflicting operations without changing the result.
    Example: 
    Schedule:
T1: R(x) W(x)
T2: R(x) W(x) R(x)
                    
Conflict Serializable:
Yes, if T2 operations can be reordered without affecting the final state.

  24. Answer: ARIES (Algorithm for Recovery and Isolation Exploiting Semantics): A recovery algorithm used to ensure database consistency after a crash.
    Steps in ARIES:
    • Analysis Phase: Identifies dirty pages and active transactions.
    • Redo Phase: Reapplies all changes to ensure updates of committed transactions are reflected.
    • Undo Phase: Reverses changes of uncommitted transactions.
    View Serializability: Ensures that a schedule can be transformed into a serial schedule by maintaining the same final state and equivalent read and write sequences.
